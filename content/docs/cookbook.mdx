These examples run against a local gateway (`python -m hatchery.core.local_dev`) or
any Hatchery gateway you point them at with `--base-url`/`--token`. They're
validation and experimentation launchpads, not fully tuned recipes for every
model family — see [Known Limitations](/docs).

## SFT + GRPO: Pig Latin

`hatchery/core/examples/train_sft.py` fine-tunes a pretrained base model to
translate English into pig latin using sentences from WikiText-2, then runs a
GRPO phase on top of the SFT checkpoint:

- **Phase 1 (SFT):** chat-template tokenization with prompt masking, batched
  training.
- **Phase 2 (GRPO):** samples 4 completions per prompt, scores them with a
  character-level edit-distance reward, computes advantages, and runs
  server-side GRPO optimization.

By default the example borrows the chat template from the matching
instruction model (for example `Qwen/Qwen2-0.5B-Instruct`) for formatting,
while the model being trained remains the base checkpoint.

```bash
python -m hatchery.core.local_dev
```

```bash
python -m hatchery.core.examples.train_sft \
  --base-url http://127.0.0.1:8420 \
  --token dev \
  --base-model Qwen/Qwen2-0.5B \
  --steps 100 \
  --rl-steps 25 \
  --batch-size 2
```

Set `--rl-steps 0` to skip the GRPO phase and run SFT only.

## Style Transfer: Pirate Chat

`hatchery/core/examples/train_pirate_style.py` is a smaller SFT-only smoke
test that fine-tunes chat-style transfer on the `winglian/pirate-ultrachat-10k`
dataset, reusing the tokenizer and label-shifting helpers from the SFT
example above.

```bash
python -m hatchery.core.examples.train_pirate_style \
  --base-url http://127.0.0.1:8420 \
  --token dev \
  --base-model Qwen/Qwen2-0.5B \
  --steps 100 \
  --batch-size 2
```

Both examples print loss, save a checkpoint, and sample from the trained
adapter when they finish.

## Custom Loss Plugin

`hatchery/core/examples/loss_plugin_example.py` shows how to register a
server-side loss function without forking core, either via the
`hatchery.losses` entry-point group or by calling `register_loss(...)`
directly. See [Extending Hatchery Core](development/extending) for the full
contract.
